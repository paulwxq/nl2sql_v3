# 🚀 NL2SQL 命令行工具 - 快速开始

## 一、安装和准备

### 1.1 确保依赖已安装
```bash
# 检查 Python 版本 (需要 3.12+)
python --version

# 安装依赖
pip install -r requirements.txt
# 或使用 uv
uv sync
```

### 1.2 配置环境变量
确保 `.env` 文件包含必要的配置：
```env
DASHSCOPE_API_KEY=your_api_key_here
POSTGRES_HOST=localhost
POSTGRES_PORT=5432
POSTGRES_DB=your_database
POSTGRES_USER=your_user
POSTGRES_PASSWORD=your_password
NEO4J_URI=bolt://localhost:7687
NEO4J_USER=neo4j
NEO4J_PASSWORD=your_password
```

### 1.3 确保服务运行
- ✅ PostgreSQL 数据库
- ✅ Neo4j 图数据库
- ✅ 网络连接正常

## 四、工具特点

### ✅ 对话式交互
- **持续对话**：像ChatGPT一样，可以连续提问
- **友好提示**：清晰的交互提示，用户体验好
- **清屏功能**：输入 `clear` 可以清屏
- **灵活退出**：支持多种退出方式（exit/quit/Ctrl+C）

### ✅ 自动化
- **自动包装输入格式**：无需手动构造复杂的 JSON
- **自动生成查询ID**：每个查询有唯一标识（时间戳+UUID）
- **自动日志记录**：所有关键步骤自动记录

### ✅ 智能化
- **问题自动解析**：理解时间、维度、指标等信息
- **Schema智能检索**：自动找到相关的表和列
- **三层验证**：语法、安全、数据库验证确保SQL正确性
- **自动重试**：验证失败时自动反馈错误重新生成（最多3次）

### ✅ 友好性
- **对话式输出**：像聊天一样自然的交互方式
- **清晰显示**：格式化的SQL，直观的成功/失败提示
- **性能统计**：显示迭代次数和执行耗时
- **温馨提示**：性能问题自动提示（如全表扫描）

## 七、故障排查

### 问题1：ModuleNotFoundError
```
ModuleNotFoundError: No module named 'src'
```

**原因**：不在项目根目录运行

**解决**：
```bash
# 切换到项目根目录
cd C:\Projects\cursor_2025h2\nl2sql_v3
python scripts/nl2sql_subgraph_cli.py "你的问题"
```

### 问题2：数据库连接失败
```
❌ SQL生成失败!
错误信息: Schema检索失败: 数据库连接失败
```

**解决步骤**：
1. 检查 PostgreSQL 是否运行：`psql -U postgres -c "SELECT 1"`
2. 检查 Neo4j 是否运行：访问 `http://localhost:7474`
3. 检查 `.env` 文件中的数据库配置
4. 测试连接：`python -c "from src.services.db.pg_client import get_pg_client; get_pg_client().test_connection()"`

### 问题3：API密钥错误
```
❌ SQL生成失败!
错误信息: 问题解析失败: API调用失败
```

**解决步骤**：
1. 检查 `.env` 文件中的 `DASHSCOPE_API_KEY`
2. 验证密钥有效性：登录阿里云控制台检查
3. 检查密钥额度是否充足

### 问题4：验证失败（表不存在）
```
❌ SQL生成失败!
错误信息: relation "public.some_table" does not exist
```

**原因**：数据库中没有对应的表

**解决**：
1. 检查数据库中是否有相关表
2. 确认表名是否正确（区分大小写）
3. 检查 Schema 是否正确（`public` vs 其他）

## 八、内部工作流程

```
用户输入问题
    ↓
[format_input] 自动包装成标准格式
    {
        "query": "...",
        "query_id": "q_20251102_143052_xxx",
        "dependencies_results": {},
        "user_query": "..."
    }
    ↓
[run_sql_generation_subgraph] 调用子图
    ↓
    ├─ question_parsing_node
    │  └─ LLM 解析问题 → 结构化 parse_result
    ↓
    ├─ schema_retrieval_node
    │  ├─ 向量检索相关表和列
    │  ├─ Neo4j 查询 JOIN 路径
    │  └─ 维度值检索和匹配
    ↓
    ├─ sql_generation_node
    │  ├─ 检索历史相似SQL
    │  ├─ 构建提示词
    │  └─ LLM 生成 SQL
    ↓
    └─ validation_node
       ├─ 第1层：语法检查
       ├─ 第2层：安全检查
       └─ 第3层：EXPLAIN 验证
          ↓
          ├─ ✅ 通过 → 返回SQL
          └─ ❌ 失败 → 重试（最多3次）
    ↓
[print_result] 格式化输出到命令行
    ↓
[logger] 记录到日志文件
```

## 九、输入格式说明

工具会自动将自然语言问题转换为以下格式：

```python
{
    "query": "请对比一下9月份京东便利和全家这两个公司的销售金额",
    "query_id": "q_20251102_143052_a1b2c3d4",  # 自动生成
    "dependencies_results": {},                 # 暂时为空
    "user_query": "请对比一下9月份京东便利和全家这两个公司的销售金额"
}
```

**字段说明**：
- `query`: 子查询文本（当前版本与 user_query 相同）
- `query_id`: 唯一查询标识（格式：`q_YYYYMMDD_HHMMSS_UUID8`）
- `dependencies_results`: 依赖查询结果（未来支持多步查询时使用）
- `user_query`: 用户原始查询
