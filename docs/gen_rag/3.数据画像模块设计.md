# 数据画像模块设计

## 1. 模块概述

### 1.1 模块定位
数据画像模块是**元数据生成模块的组成部分**，在元数据提取和列统计完成后，对表和字段进行业务语义分析和分类，为后续的关系发现、SQL生成提供更丰富的语义信息。

**集成方式**：
- 数据画像功能**集成在第一阶段（元数据生成）中**
- 在采样数据和统计信息生成后立即执行
- 先生成列画像，再基于列画像生成表画像
- 一次运行，输出完整的元数据JSON

### 1.2 核心目标
- 基于列的统计特征和命名模式，对列进行语义角色分类
- 基于列的特征组合，对表进行类型分类（事实表/维表/桥接表等）
- 提供清晰的推断依据和置信度评分
- 为SQL生成提供列的使用建议（可聚合、可分组、可筛选等）

### 1.3 处理时机与依赖

**在元数据生成流程中的位置**：
```
元数据生成流程：
  1. 提取表结构信息
  2. 提取约束信息（主键、外键、唯一约束、索引）
  3. 识别逻辑主键
  4. 采样数据
  5. 计算列统计信息（statistics）
  ↓
  6. 生成列画像（column_profile）← 数据画像：步骤1
  ↓
  7. 生成表画像（table_profile）← 数据画像：步骤2
  ↓
  8. 保存完整的元数据JSON
```

**输入依赖**：
- 列的基本信息（column_name, data_type, is_nullable等）
- 列的统计信息（statistics：count, unique_count, null_rate等）
- 表的约束信息（primary_keys, foreign_keys, unique_constraints, indexes）
- 逻辑主键识别结果（logical_keys）
  - **注意**：`logical_keys` 中仅包含最小候选键，已自动过滤掉 superkeys（参见《2.数据库元数据生成模块设计》第 2.3.2 节）

**输出结果**：
- 在每个列的JSON中添加 `column_profile` 字段
- 在表的JSON中添加 `table_profile` 字段
- 最终生成包含完整画像信息的元数据JSON文件

## 2. 列画像设计

### 2.1 列的语义角色分类

列画像将每个字段分为5种**互斥**的语义角色：

| 角色 | 说明 | 典型场景 |
|------|------|----------|
| **identifier** | 标识符列 | ID、编码、键值 |
| **metric** | 指标/度量列 | 金额、数量、计数 |
| **datetime** | 日期时间列 | 日期、时间戳 |
| **enum** | 枚举列 | 状态、类型、级别 |
| **attribute** | 属性列 | 名称、描述、地址 |

### 2.2 列的结构标志

除了语义角色，还为每个列标注**独立的**结构特征标志：

| 标志 | 说明 | 来源 |
|------|------|------|
| **is_primary_key** | 是否为主键 | 约束元数据 或 逻辑主键推断 |
| **is_foreign_key** | 是否为外键 | 约束元数据 |
| **is_unique** | 是否唯一（数据层面） | 统计信息 |
| **is_unique_constraint** | 是否有唯一约束 | 约束元数据 |
| **is_indexed** | 是否有索引 | 索引元数据 |
| **is_nullable** | 是否可空 | 列定义 |

**重要说明**：
- 语义角色与结构标志**相互独立**
- 一个列可以既是"identifier"又是"foreign_key"
- 一个列可以既是"metric"又是"indexed"

### 2.3 列画像JSON结构

```json
{
  "column_name": "store_id",
  "data_type": "integer",
  "comment": "店铺ID（外键）",
  
  "statistics": {
    "count": 144,
    "unique_count": 9,
    "null_count": 0,
    "null_rate": 0.0,
    "uniqueness": 0.0625,
    "value_distribution": {...}
  },
  
  "column_profile": {
    "semantic_role": "identifier",
    "semantic_confidence": 0.9,
    
    "structure_flags": {
      "is_primary_key": false,
      "is_foreign_key": true,
      "is_unique": false,
      "is_unique_constraint": false,
      "is_indexed": true,
      "is_nullable": false
    },
    
    "identifier_info": {
      "naming_pattern": "table_id",
      "reference_table": "dim_store"
    },
    
    "inference_basis": [
      "name_pattern_match",
      "foreign_key_metadata"
    ]
  }
}
```

### 2.4 列语义角色识别规则

#### 2.4.1 Identifier（标识符列）

**判断规则**（按优先级）：

**前置条件（规则0）**：数据类型白名单检查
- 必须通过类型白名单，才会进入后续判断
- 允许的类型：整数、字符串、UUID、无小数的 numeric
- 排除的类型：json、jsonb、array、bytea、geometry 等复杂类型
- 配置路径：`sampling.identifier_detection.allowed_data_types`

**主要规则**（满足任一即可）：
1. **物理约束**（100%确定）：有 PRIMARY KEY / FOREIGN KEY / UNIQUE 约束
2. **统计特征**：唯一性 > 0.95 且非空率 > 0.80
3. **命名模式**：
   - 字段名匹配模式：`*_id`, `*_key`, `*_code`, `*_no`, `*_number`
   - 字段名完全匹配：`id`, `key`, `code`, `no`
   - 且唯一性 > 0.05（排除枚举）

**示例**：
- `store_id`, `product_key`, `order_no`, `company_code`
- `id`, `pk`

**置信度**：
- 物理约束：1.0
- 统计特征：0.95
- 命名模式：0.9（完全匹配）或 0.85（后缀匹配）

#### 2.4.2 Metric（指标列）

**判断规则**（同时满足）：
1. 数据类型为数值型：`numeric`, `decimal`, `integer`, `bigint`, `float`, `double`, `real`
2. 字段名匹配度量模式：
   - 金额类：`amount`, `total`, `sum`, `price`, `cost`, `revenue`, `sales`, `profit`
   - 数量类：`count`, `qty`, `quantity`, `num`, `number`, `volume`
   - 比率类：`rate`, `ratio`, `percent`, `percentage`

**示例**：
- `amount`, `total_sales`, `order_count`, `unit_price`, `profit_margin`

**排除**：
- 标识符列（即使是数值型）
- 日期时间列

**置信度**：
- 类型+命名双匹配：0.95
- 仅类型匹配：0.7

#### 2.4.3 DateTime（日期时间列）

**判断规则**（满足其一）：
1. 数据类型匹配：`date`, `datetime`, `timestamp`, `timestamptz`, `time`
2. 字段名匹配+数值型：`year`, `month`, `quarter`, `day`, `hour`（整数类型）

**日期粒度识别**：
从字段名推断：
- `*_year` → year
- `*_quarter` → quarter
- `*_month` → month
- `*_week` → week
- `*_day`, `*_date` → day
- `*_hour` → hour
- `*_minute` → minute

**示例**：
- `created_at`, `updated_at`, `date_day`, `order_date`, `timestamp`
- `year`, `month`（整数类型）

**置信度**：
- 类型匹配：1.0
- 命名匹配：0.85

#### 2.4.4 Enum（枚举列）

**判断规则**（同时满足）：
1. `unique_count` ≤ 阈值（默认10，可配置）
2. `statistics` 中包含 `value_distribution`

**枚举基数分级**：
- **低基数**（low）：unique_count ≤ 10
- **中基数**（medium）：10 < unique_count ≤ 50

**示例**：
- `status` (正常, 禁用, 删除)
- `type` (A, B, C, D)
- `level` (1, 2, 3, 4, 5)
- `city_name` (广州市, 深圳市, 南京市, ...)

**排除**：
- 已识别为 identifier 的列
- 已识别为 datetime 的列

**置信度**：
- 基数 ≤ 10：0.95
- 基数 10-20：0.85
- 基数 20-50：0.75

#### 2.4.5 Attribute（属性列）

**判断规则**：
- 兜底分类：不属于上述4类的所有列

**典型字段**：
- 名称类：`*_name`, `title`, `label`
- 描述类：`description`, `remark`, `comment`, `note`
- 地址类：`address`, `location`
- 其他文本：`text`, `content`, `message`

**置信度**：0.7

### 2.5 列结构标志识别规则

#### 2.5.1 is_primary_key

**判断逻辑**（优先级从高到低）：

```python
# 优先级1：从约束元数据获取
if column_name in metadata.primary_keys[*].columns:
    is_primary_key = True
    pk_source = "constraint"

# 优先级2：从逻辑主键推断（仅当无约束主键时）
# 注意：logical_keys[0] 是置信度最高的最小候选键（已过滤 superkeys）
elif metadata.primary_keys is empty and metadata.logical_keys:
    if column_name in metadata.logical_keys[0].columns:
        is_primary_key = True
        pk_source = "logical"
        pk_confidence = metadata.logical_keys[0].confidence_score

# 否则
else:
    is_primary_key = False
```

**补充信息**：
```json
"primary_key_info": {
  "source": "logical",          // "constraint" | "logical"
  "confidence": 0.95,            // 仅 logical 有此字段
  "is_single_column": true,      // 是否单列主键
  "composite_columns": null      // 复合主键的其他列
}
```

#### 2.5.2 is_foreign_key

**判断逻辑**：

```python
# 仅从约束元数据获取
for fk in metadata.foreign_keys:
    if column_name in fk.source_columns:
        is_foreign_key = True
        target_table = fk.target_table
        target_columns = fk.target_columns
```

**注意**：
- **不进行推断**：即使字段名为 `xxx_id`，也不自动判定为外键
- 必须有数据库约束定义

**补充信息**：
```json
"foreign_key_info": {
  "target_schema": "public",
  "target_table": "dim_store",
  "target_columns": ["store_id"],
  "on_delete": "NO ACTION",
  "on_update": "NO ACTION"
}
```

#### 2.5.3 is_unique 和 is_unique_constraint

**is_unique判断逻辑**（数据层面）：

```python
# 从统计信息判断
if statistics.uniqueness == 1.0:
    is_unique = True
else:
    is_unique = False
```

**is_unique_constraint判断逻辑**（约束层面）：

```python
# 从唯一约束元数据
for uc in metadata.unique_constraints:
    if column_name in uc.columns:
        is_unique_constraint = True
```

**说明**：
- `is_unique`: 数据层面的唯一性，基于统计分析
- `is_unique_constraint`: 数据库约束层面的唯一性，基于元数据

#### 2.5.4 is_indexed

**判断逻辑**：

```python
for index in metadata.indexes:
    if column_name in index.columns:
        is_indexed = True
```

**补充信息**：
```json
"index_info": {
  "index_name": "idx_store_company",
  "index_type": "btree",
  "is_unique": false,
  "position": 1  // 在索引中的位置
}
```

#### 2.5.5 is_nullable

**判断逻辑**：
```python
is_nullable = column_info.is_nullable
```

直接从列定义读取。

### 2.6 列画像识别流程

```
输入：ColumnInfo + Statistics + Metadata
  ↓
┌─────────────────────────────────┐
│ 1. 识别语义角色（互斥，5选1）     │
├─────────────────────────────────┤
│ a. datetime   ← 数据类型优先      │
│ b. identifier ← 命名模式          │
│ c. enum       ← 唯一值数量        │
│ d. metric     ← 数值+命名         │
│ e. attribute  ← 兜底              │
└─────────────────────────────────┘
  ↓
┌─────────────────────────────────┐
│ 2. 识别结构标志（独立，多个）     │
├─────────────────────────────────┤
│ a. is_primary_key        ← 约束/逻辑  │
│ b. is_foreign_key        ← 约束       │
│ c. is_unique             ← 统计       │
│ d. is_unique_constraint  ← 约束       │
│ e. is_indexed            ← 索引元数据 │
│ f. is_nullable           ← 列定义     │
└─────────────────────────────────┘
  ↓
┌─────────────────────────────────┐
│ 3. 生成补充信息                  │
├─────────────────────────────────┤
│ - identifier_info                │
│ - metric_info                    │
│ - datetime_info                  │
│ - enum_info                      │
│ - primary_key_info               │
│ - foreign_key_info               │
│ - index_info                     │
└─────────────────────────────────┘
  ↓
输出：ColumnProfile
```

## 3. 表画像设计

### 3.1 表类型分类

表画像将每个表分为以下类型：

| 表类型 | 说明 | 典型特征 |
|--------|------|----------|
| **fact** | 事实表 | 有度量列，多个标识符，有日期维度 |
| **dim** | 维表 | 有主键，属性列为主，无度量列 |
| **bridge** | 桥接表 | 主要是外键（≥2个），列数少，无度量 |
| **unknown** | 未知类型 | 不符合上述任何模式 |

### 3.2 表画像JSON结构

```json
{
  "schema_name": "public",
  "table_name": "fact_store_sales_day",
  "table_type": "table",
  "comment": "店铺销售日流水事实表",
  
  "columns": [...],
  "primary_keys": [...],
  "foreign_keys": [...],
  
  "table_profile": {
    "table_category": "fact",
    "confidence": 0.95,
    
    "column_statistics": {
      "total_columns": 4,
      "identifier_count": 2,
      "metric_count": 1,
      "datetime_count": 1,
      "enum_count": 0,
      "attribute_count": 0,
      "primary_key_count": 0,
      "foreign_key_count": 0
    },
    
    "key_columns": {
      "primary_keys": [],
      "logical_keys": ["store_id", "date_day", "product_type_id"],
      "foreign_keys": [],
    },
    
    "fact_table_info": {
      "grain": ["store_id", "date_day", "product_type_id"],
      "metrics": ["amount"],
      "dimensions": ["store_id", "date_day", "product_type_id"],
      "time_dimension": "date_day"
    },
    
    "inference_basis": [
      "has_metric_columns",
      "has_multiple_identifiers",
      "has_datetime_column",
      "naming_pattern_fact"
    ]
  }
}
```

### 3.3 表类型识别规则

#### 3.3.1 Fact（事实表）

**判断规则**（同时满足多个条件）：

**必要条件**（至少满足2个）：
1. 有度量列（metric）≥ 1
2. 有外键或标识符列 ≥ 2
3. 有日期时间列 ≥ 1

**辅助条件**（加分项）：
- 表名包含：`fact_*`, `*_fact`, `agg_*`, `sum_*`
- 表注释包含：事实表、流水、汇总、明细
- 有复合逻辑主键（多个字段组合）

**排除条件**：
- 无度量列

**示例**：
```json
{
  "table_name": "fact_store_sales_day",
  "columns": [
    {"name": "store_id", "role": "identifier"},
    {"name": "date_day", "role": "datetime"},
    {"name": "product_type_id", "role": "identifier"},
    {"name": "amount", "role": "metric"}
  ],
  "table_category": "fact",
  "confidence": 0.95
}
```

**置信度计算**：
```python
confidence = 0.6  # 基础分

# 必要条件
if has_metric: confidence += 0.15
if has_multiple_identifiers: confidence += 0.1
if has_datetime: confidence += 0.1

# 辅助条件
if matches_naming_pattern: confidence += 0.05
if has_composite_key: confidence += 0.05

# 上限
confidence = min(confidence, 0.98)
```

#### 3.3.2 Dim（维表）

**判断规则**：

**必要条件**（同时满足）：
1. 有主键（约束或逻辑主键）
2. 度量列 = 0
3. 标识符列 ≥ 1

**辅助条件**：
- 表名包含：`dim_*`, `*_dim`, `d_*`
- 表注释包含：维表、维度、字典
- 属性列占比 > 50%

**示例**：
```json
{
  "table_name": "dim_store",
  "columns": [
    {"name": "store_id", "role": "identifier", "is_primary_key": true},
    {"name": "store_name", "role": "attribute"},
    {"name": "company_id", "role": "identifier", "is_foreign_key": true},
    {"name": "region_id", "role": "identifier", "is_foreign_key": true}
  ],
  "table_category": "dim",
  "confidence": 0.9
}
```

**置信度计算**：
```python
confidence = 0.7  # 基础分

if has_primary_key: confidence += 0.15
if no_metric_columns: confidence += 0.1
if matches_naming_pattern: confidence += 0.05

confidence = min(confidence, 0.95)
```

#### 3.3.3 Bridge（桥接表）

**判断规则**（同时满足）：

1. 外键数量 ≥ 2
2. 列数 ≤ 5
3. 度量列 = 0 或 只有权重/比例类度量

**典型结构**：
- 外键1 + 外键2
- 外键1 + 外键2 + 权重
- 外键1 + 外键2 + 生效日期 + 失效日期

**示例**：

```json
{
  "table_name": "bridge_account_customer",
  "columns": [
    {"name": "account_id", "role": "identifier", "is_foreign_key": true},
    {"name": "customer_id", "role": "identifier", "is_foreign_key": true},
    {"name": "weight", "role": "metric"}
  ],
  "table_category": "bridge",
  "confidence": 0.85
}
```

### 3.4 表画像识别流程

```
输入：TableMetadata（含ColumnProfile）
  ↓
┌─────────────────────────────────┐
│ 1. 统计列特征                    │
├─────────────────────────────────┤
│ - 统计各语义角色的列数量         │
│ - 统计结构标志的列数量           │
│ - 识别关键列（主键、外键）       │
└─────────────────────────────────┘
  ↓
┌─────────────────────────────────┐
│ 2. 命名模式匹配                  │
├─────────────────────────────────┤
│ - 表名匹配（fact_*, dim_*）      │
│ - 表注释关键词提取               │
└─────────────────────────────────┘
  ↓
┌─────────────────────────────────┐
│ 3. 表类型判断（优先级顺序）      │
├─────────────────────────────────┤
│ a. bridge       ← 外键为主       │
│ b. fact         ← 有度量         │
│ c. dim          ← 有主键无度量   │
│ d. unknown      ← 兜底           │
└─────────────────────────────────┘
  ↓
┌─────────────────────────────────┐
│ 4. 生成补充信息                  │
├─────────────────────────────────┤
│ - fact_table_info                │
│ - dim_table_info                 │
│ - bridge_table_info              │
└─────────────────────────────────┘
  ↓
输出：TableProfile
```

## 4. 配置文件设计

### 4.1 配置文件结构

画像配置集成在元数据生成配置文件中：

```yaml
# configs/metaweave/metadata_config.yaml

# ... 前面的数据库连接等 ...

# 数据采样配置
sampling:
  enabled: true
  sample_size: 1000
  
  # 标识符识别配置
  identifier_detection:
    # 数据类型白名单：只有这些类型才可能被识别为 identifier
    # 使用白名单而非黑名单，确保安全性和可预测性
    allowed_data_types:
      - integer
      - bigint
      - smallint
      - serial
      - varchar
      - char
      - uuid
      - numeric   # 需要 scale=0
      - decimal   # 需要 scale=0
    # 统计阈值
    high_uniqueness_threshold: 0.95
    min_non_null_rate: 0.80
    low_uniqueness_threshold: 0.05

# 数据画像配置
profiling:
  enabled: true  # 是否启用数据画像功能
  
  # 列画像配置
  column_profiling:
    enabled: true
    
    # 枚举列阈值
    enum_threshold: 10
    
    # 度量列命名模式
    metric_patterns:
      - amount
      - total
      - sum
      - count
      - qty
      - quantity
      - price
      - cost
      - revenue
      - sales
      - profit
      - num
      - number
      - volume
      - rate
      - ratio
      - percent
    
    # 标识符命名模式
    identifier_patterns:
      - _id$
      - _key$
      - _code$
      - _no$
      - _number$
      - ^id$
      - ^key$
    
    # 日期时间类型
    datetime_types:
      - date
      - datetime
      - timestamp
      - timestamptz
      - time
  
  # 表画像配置
  table_profiling:
    enabled: true
    
    # 事实表识别
    fact_table:
      min_metric_columns: 1
      min_dimension_columns: 2
      require_datetime: true
      naming_patterns:
        - ^fact_
        - _fact$
        - ^agg_
        - ^sum_
      comment_keywords:
        - 事实表
        - 流水
        - 明细
        - 汇总
    
    # 维表识别
    dim_table:
      require_primary_key: true
      max_metric_columns: 0
      naming_patterns:
        - ^dim_
        - _dim$
        - ^d_
      comment_keywords:
        - 维表
        - 维度
        - 字典
    
    # 桥接表识别
    bridge_table:
      min_foreign_keys: 2
      max_columns: 5
      max_metric_columns: 1
      naming_patterns:
        - ^bridge_
        - _bridge$
```

### 6.2 列画像处理流程

```python
def profile_column(column: ColumnInfo, statistics: dict, metadata: TableMetadata) -> ColumnProfile:
    """处理单个列的画像"""
    
    # 1. 识别语义角色（5种：identifier/metric/datetime/enum/attribute）
    semantic_role = classify_semantic_role(column, statistics)
    
    # 2. 识别结构标志
    structure_flags = classify_structure_flags(column.column_name, metadata, statistics)
    
    # 3. 生成补充信息
    profile = ColumnProfile(
        semantic_role=semantic_role,
        semantic_confidence=calculate_semantic_confidence(semantic_role, column, statistics),
        structure_flags=structure_flags
    )
    
    # 根据语义角色生成详细信息
    if semantic_role == 'identifier':
        profile.identifier_info = generate_identifier_info(column)
    elif semantic_role == 'metric':
        profile.metric_info = generate_metric_info(column, statistics)
    elif semantic_role == 'datetime':
        profile.datetime_info = generate_datetime_info(column)
    elif semantic_role == 'enum':
        profile.enum_info = generate_enum_info(column, statistics)
    # attribute 角色无需额外信息
    
    # 根据结构标志生成详细信息
    if structure_flags.is_primary_key:
        profile.primary_key_info = generate_primary_key_info(column, metadata)
    if structure_flags.is_foreign_key:
        profile.foreign_key_info = generate_foreign_key_info(column, metadata)
    if structure_flags.is_indexed:
        profile.index_info = generate_index_info(column, metadata)
    
    return profile
```

### 6.3 表画像处理流程

```python
def profile_table(metadata: TableMetadata, column_profiles: List[ColumnProfile]) -> TableProfile:
    """处理单张表的画像"""
    
    # 1. 统计列特征
    column_stats = calculate_column_statistics(column_profiles)
    
    # 2. 提取关键列
    key_columns = extract_key_columns(metadata, column_profiles)
    
    # 3. 识别表类型（3种：fact/dim/bridge）
    table_category = classify_table_type(
        metadata, 
        column_stats, 
        key_columns, 
        column_profiles
    )
    
    # 4. 生成表画像
    profile = TableProfile(
        table_category=table_category,
        confidence=calculate_table_confidence(table_category, metadata, column_stats),
        column_statistics=column_stats,
        key_columns=key_columns
    )
    
    # 5. 根据表类型生成补充信息
    if table_category == 'fact':
        profile.fact_table_info = generate_fact_table_info(metadata, column_profiles)
    elif table_category == 'dim':
        profile.dim_table_info = generate_dim_table_info(metadata, column_profiles)
    elif table_category == 'bridge':
        profile.bridge_table_info = generate_bridge_table_info(metadata, column_profiles)
    
    return profile
```

## 8. 命令行接口

数据画像功能集成在元数据生成命令中，通过配置文件控制是否启用：

```bash
# 生成元数据（默认包含数据画像）
python -m src.metaweave.cli.main generate --config configs/metaweave/metadata_config.yaml --schemas public

# 生成元数据（明确指定启用画像）
python -m src.metaweave.cli.main generate --config configs/metaweave/metadata_config.yaml --schemas public --enable-profiling

# 生成元数据（禁用画像，仅生成基础元数据）
python -m src.metaweave.cli.main generate --config configs/metaweave/metadata_config.yaml --schemas public --no-profiling
```

**配置文件控制**：

```yaml
# configs/metaweave/metadata_config.yaml
profiling:
  enabled: true  # 设为 false 可禁用画像功能
  column_profiling:
    enabled: true  # 可单独控制列画像
  table_profiling:
    enabled: true  # 可单独控制表画像
```
