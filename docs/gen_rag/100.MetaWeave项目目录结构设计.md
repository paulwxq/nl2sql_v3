# MetaWeave 项目目录结构设计

## 文档说明

本文档定义了 MetaWeave RAG 平台的完整目录结构规划。MetaWeave 作为 NL2SQL 项目的独立子模块，主要负责从 PostgreSQL 数据库中抽取元数据、生成数据画像、发现表关系，并将结果导入图数据库和向量数据库。

**设计原则：**
- MetaWeave 在 `src/` 目录下作为独立模块，保持相对独立性
- 遵循 Python src layout 标准实践
- 模块化设计，便于维护和扩展
- 配置、脚本、输出分层管理
- 可复用共享服务，避免重复开发
- 将来如需独立运行，可整体抽离

## 目录说明

### 1. 核心代码结构 (`src/metaweave/`)

#### 1.1 core/ - 核心功能模块
MetaWeave 的核心业务逻辑，按功能划分为5个子模块：

- **metadata/** - 元数据生成
  - 从PostgreSQL提取表结构、字段、约束、索引等信息
  - 基于LLM生成缺失的注释
  - 识别逻辑主键
  - 输出DDL、Markdown、JSON格式

- **profiling/** - 数据画像
  - 表画像：分类为fact/dim/bridge
  - 列画像：唯一度、空值率、枚举度、值范围等统计

- **relationship/** - 关系评估
  - 生成关系候选（单列/复合键）
  - 计算包含率、Jaccard系数、置信度
  - 决策引擎选择最终关系

- **domain/** - 数据域与SQL生成
  - 主题域分类
  - 模拟业务需求
  - 生成Question/SQL样例

- **export/** - 导出
  - 生成Cypher语句
  - 加载到Neo4j图数据库
  - 加载到Milvus向量数据库

#### 1.2 services/ - 服务层
封装外部依赖和通用服务：
- `llm_service.py`: LLM接口封装（支持OpenAI、Claude等）
- `cache_service.py`: 缓存管理

#### 1.3 utils/ - 工具层
通用工具函数：
- `file_utils.py`: 文件读写、目录管理
- `sql_templates.py`: SQL查询模板
- `data_utils.py`: 数据处理、转换
- `validator.py`: 数据验证

#### 1.4 cli/ - 命令行接口
提供友好的命令行工具：
- `main.py`: 统一入口，支持子命令
- 各功能模块的独立CLI

### 2. 配置管理 (`configs/metaweave/`)

集中管理所有配置文件：
- 数据库连接配置
- LLM配置（API Key、模型选择、参数）
- 采样策略配置
- 阈值和决策规则配置
- 输出格式配置
- 日志级别配置

### 3. 脚本目录 (`scripts/metaweave/`)

提供可直接执行的Python脚本：
- 各模块独立运行脚本
- `run_pipeline.py`: 一键运行完整流程
- 支持命令行参数传递

### 4. 输出目录 (`output/metaweave/`)

分模块组织输出文件：
- 便于查看各阶段的输出结果
- 便于调试和验证
- 可配置输出路径

### 5. 缓存目录 (`cache/metaweave/`)

存储中间计算结果：
- LLM生成的注释缓存（避免重复调用）
- 元数据缓存（支持增量更新）
- 关系计算缓存（提升性能）

### 6. 日志目录 (`logs/metaweave/`)

记录运行日志：
- 各模块独立日志文件
- 便于问题排查
- 可配置日志级别和格式

### 7. 测试目录 (`tests/metaweave/`)

完整的测试覆盖：
- 单元测试：测试各个类和函数
- 集成测试：测试完整流程
- 使用pytest框架

## 运行方式

### 方式1：通过CLI命令运行（推荐）

```bash
# 运行元数据生成
python -m src.metaweave.cli.main metadata --config configs/metaweave/metadata_config.yaml

# 运行数据画像
python -m src.metaweave.cli.main profiling --config configs/metaweave/profiling_config.yaml

# 运行关系评估
python -m src.metaweave.cli.main relationship --config configs/metaweave/relationship_config.yaml

# 运行数据域分类和SQL生成
python -m src.metaweave.cli.main domain --config configs/metaweave/domain_config.yaml

# 运行导出
python -m src.metaweave.cli.main export --config configs/metaweave/export_config.yaml

# 运行完整流程
python -m src.metaweave.cli.main pipeline --config-dir configs/metaweave/
```

### 方式2：通过脚本运行

```bash
# 进入项目根目录
cd /path/to/nl2sql_v3

# 运行单个模块
python scripts/metaweave/run_metadata_generation.py --config configs/metaweave/metadata_config.yaml

# 运行完整流程
python scripts/metaweave/run_pipeline.py --config-dir configs/metaweave/

# 指定特定的schema和表
python scripts/metaweave/run_metadata_generation.py \
  --config configs/metaweave/metadata_config.yaml \
  --schemas public,schema2 \
  --tables users,orders

# 增量更新模式
python scripts/metaweave/run_metadata_generation.py \
  --config configs/metaweave/metadata_config.yaml \
  --incremental

# 调试模式
python scripts/metaweave/run_metadata_generation.py \
  --config configs/metaweave/metadata_config.yaml \
  --debug
```

### 方式3：在代码中引用

```python
# 在NL2SQL模块中引用MetaWeave
from src.metaweave.core.metadata import MetadataGenerator
from src.metaweave.core.export import VectorLoader

# 初始化
generator = MetadataGenerator(config_path='configs/metaweave/metadata_config.yaml')

# 生成元数据
result = generator.generate(schemas=['public'], tables=None)

# 加载到向量库
loader = VectorLoader(config_path='configs/metaweave/export_config.yaml')
loader.load_metadata(result)
```

## 配置文件示例

### metadata_config.yaml

```yaml
# 数据库连接
database:
  host: localhost
  port: 5432
  database: your_database
  user: postgres
  password: ${DB_PASSWORD}
  schemas:
    - public
  exclude_tables:
    - temp_*

# LLM配置
llm:
  provider: openai
  model: gpt-4
  api_key: ${LLM_API_KEY}
  temperature: 0.3

# 采样配置
sampling:
  sample_size: 1000
  method: limit

# 输出配置
output:
  output_dir: output/metaweave/metadata
  formats:
    - ddl
    - markdown
    - json
```

```
  - 数据采样
  - 表分类
  - 列统计分析
  → 输出: 画像MD
      ↓
3. relationship 模块
  - 生成候选关系
  - 计算评分
  - 决策最终关系
  → 输出: 关系MD
      ↓
4. domain 模块
  - 主题域分类
  - 生成SQL样例
  → 输出: SQL样例MD
      ↓
5. export 模块
  - 生成Cypher
  - 加载到Neo4j
  - 加载到Milvus
  → 输出: RAG数据就绪
      ↓
NL2SQL 模块使用
```
