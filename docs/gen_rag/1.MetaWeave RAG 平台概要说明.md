# MetaWeave RAG 平台概要说明

## 项目定位目标

自动化的从 PostgreSQL 数据库中抽取元数据，生成数据画像，发现表之间的关联关系，并将结果导入到图数据库和向量数据库，作为NL2SQL模块的RAG数据。

#### 主要抽取过程和输出

1.生成数据库元数据：连接到PostgreSQL数据库，获取表结构，字段名，字段类型，字段注释，主键、外键、唯一约束、索引信息等。如果从数据库中读取元数据中没有表名注释，字段注释，则需要调用LLM，结合样本数据为表名或字段名生成注释。结合样本数据查找表的逻辑主键或唯一约束。同时生成表定义信息的ddl文件和md文件。

2.数据采样与画像：
基于配置文件中定义的参数，抽取前N条数据，进行数据采样与画像。
- 表画像，基于表的抽样数据，对表进行分类，比如 fact(事实表、汇总表), dim (维表、实体表、枚举表、自关联维表)，bridge(桥接表)。
- 列画像，分析每个表的字段，对抽样数据进行统计分析（唯一度、样本值、空值率、值范围、枚举型的离散度）。
输出结果为md文件。

3.关系评估和决策：对于缺少主键和外键的情况，依据列名相似度、类型兼容性、模式匹配等规则生成单列和复合键关系候选。通过计算关联字段的包含率、Jaccard系数、唯一度并合成综合评分，同时标注置信度等级。最后，根据配置的阈值和决策规则挑选最终关系，支持复合关系优先、子集抑制等策略。
把关联信息输出为md文件。

4.数据域与SQL样例生成：基于表和字段的数据画像，以及表的元数据，对当前表分类到不同的主题域（主题域的定义在配置文件中定义，同一张表可能属于多个主题域），然后在每个主题下模拟业务需求，生成样例SQL(Question/SQL) 文件。输出结果为md文件。


5.最终输出结果：
	- 生成 Cypher 语句，落地成文件。
	- 生成表的元数据的md文件。
	- 从维度表、对照表加载数据到向量表中。

6.加载到图数据库：把Cypher 文件加载到图数据库，包含表节点、关系边、度量指标、属性等。

7.加载元数据到向量数据库：把数据表的md文件加载到向量数据库，对指定字段生成embedding字段。

8.加载样例SQL(Question/SQL)：把样例SQL文件加载到向量数据库，对指定字段生成embedding字段。
