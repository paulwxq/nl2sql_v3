# dim_value 维度值向量化加载使用手册

## 文档概述

本文档详细说明如何使用 `dim_value` 加载器将维度表的值向量化并加载到 Milvus 向量数据库中，用于 NL2SQL 场景中的值匹配和语义检索。

---

## 命令语法

```bash
python -m src.metaweave.cli.main load \
  --type dim_value \
  --config configs/metaweave/loader_config.yaml \
  [--clean] \
  [--debug]
```

---

## 参数说明

### 必需参数

| 参数 | 说明 | 示例 |
|------|------|------|
| `--type dim_value` | 指定加载器类型为维度值加载器 | `--type dim_value` |
| `--config <path>` | 指定 loader_config.yaml 配置文件路径 | `--config configs/metaweave/loader_config.yaml` |

### 可选参数

| 参数 | 说明 | 默认值 | 使用场景 |
|------|------|--------|----------|
| `--clean` | 删除并重建 Milvus Collection（清空所有已有数据） | 不启用 | 首次加载、完全重建、数据损坏恢复 |
| `--debug` | 启用调试级别日志输出 | INFO 级别 | 排查问题、开发调试 |

### `--clean` 参数详解

**⚠️ 重要警告**: `--clean` 参数会删除 Milvus 中的 `dim_value_embeddings` Collection 及其所有数据，此操作不可逆！

#### 执行流程对比

**不带 `--clean` 参数**:
```
1. 检查 Collection 是否存在
2. 如存在 → 直接加载并追加数据（⚠️ 会造成数据重复）
3. 如不存在 → 自动创建 Collection 并加载数据
```

**带 `--clean` 参数**:
```
1. 检查 Collection 是否存在
2. 如存在 → 删除 Collection（包括所有数据）
3. 重新创建 Collection
4. 创建 HNSW 索引
5. 加载数据
```

#### 使用建议

| 场景 | 是否使用 --clean | 说明 |
|------|------------------|------|
| 首次加载 | ❌ 不需要 | 自动创建 Collection，无数据冲突 |
| 完全重建 | ✅ 必须 | 避免新旧数据混合 |
| 增量更新 | ⚠️ 暂不支持 | 当前版本会追加重复数据，建议使用 `--clean` |
| 修复错误数据 | ✅ 必须 | 清除错误数据并重新加载 |

在运行加载命令前，请确保以下条件已满足：

### 1️⃣ 配置文件准备

#### (1) 生成 `dim_tables.yaml` 配置文件

```bash
python -m src.metaweave.cli.main dim_config --generate
```

**执行后输出**:
```

📊 识别到 3 个维度表：
  - public.dim_company
  - public.dim_store
  - public.dim_product_type
⚠️  请手工填写 embedding_col 字段（要向量化的列名）
📝 示例：
    public.dim_company:
      embedding_col: your_text_column  # 替换为实际列名
```

#### (2) 手工配置 `embedding_col` 字段

编辑 `configs/metaweave/dim_tables.yaml`，为每个维度表指定要向量化的文本列：

```yaml
# 维度表加载配置
# 说明：此文件由 dim_config --generate 自动生成维表列表，需人工填写 embedding_col

tables:
  public.dim_company:
    embedding_col: company_name  # ✅ 手工填写实际列名
  public.dim_store:
    embedding_col: store_name    # ✅ 手工填写实际列名
  public.dim_product_type:
    embedding_col: type_name     # ✅ 手工填写实际列名
```

**选择原则**:
- 选择包含语义信息的文本列（如名称、描述）
- 避免选择 ID、代码等标识符列
- 避免选择枚举值或布尔值列

## 自动创建机制

DimValueLoader 实现了 **Convention over Configuration** 设计理念，自动创建所需资源：

### 自动创建的资源

| 资源 | 自动创建条件 | 说明 |
|------|--------------|------|
| **Milvus Database** | 不存在时 | 使用 `metadata_config.yaml` 中的 `vector_database.providers.milvus.database` |
| **Milvus Collection** | 不存在时 | 名称: `dim_value_embeddings` |
| **HNSW Index** | Collection 无索引时 | 自动为 `embedding` 字段创建索引 |

### Collection Schema

自动创建的 Collection 包含以下字段：

| 字段名 | 类型 | 说明 | 示例 |
|--------|------|------|------|
| `id` | VARCHAR(64) | 主键，格式: `{schema}:{table}:{col}:{value_hash}` | `public:dim_company:company_name:abc123` |
| `table_name` | VARCHAR(128) | 完整表名 `{schema}.{table}` | `public.dim_company` |
| `col_name` | VARCHAR(128) | 列名 | `company_name` |
| `col_value` | VARCHAR(512) | 实际文本值 | `阿里巴巴集团` |
| `embedding` | FLOAT_VECTOR(1024) | 向量化表示（通义千问 text-embedding-v3） | `[0.123, -0.456, ...]` |
| `update_ts` | VARCHAR(32) | 更新时间戳 | `2025-12-11T10:30:00Z` |

### Index 配置

自动创建的 HNSW 索引参数：

```python
{
    "index_type": "HNSW",
    "metric_type": "COSINE",  # 余弦相似度
    "params": {
        "M": 16,              # HNSW 参数：每个节点的邻居数
        "efConstruction": 256 # HNSW 参数：构建时的搜索深度
    }
}
```

## 常见问题与排查

### ❌ 错误 1: dim_tables.yaml 不存在

**错误信息**:
```
❌ 验证失败: dim_tables.yaml 文件不存在
路径: configs/metaweave/dim_tables.yaml
```

**解决方法**:
```bash
python -m src.metaweave.cli.main dim_config --generate
```

---

### ❌ 错误 2: embedding_col 未配置

**错误信息**:
```
❌ 验证失败: 表 public.dim_company 缺少 embedding_col 配置
```

**解决方法**:
编辑 `configs/metaweave/dim_tables.yaml`，为该表添加 `embedding_col` 字段。

---

### ❌ 错误 3: Milvus 连接失败

**错误信息**:
```
❌ Milvus 连接失败（已重试 3 次）: Connection refused
```

**排查步骤**:
1. 检查 Milvus 是否运行: `docker ps | grep milvus`
2. 检查端口是否监听: `netstat -an | grep 19530`
3. 检查 `.env` 中的 `MILVUS_HOST` 和 `MILVUS_PORT`

---

### ❌ 错误 4: PostgreSQL 连接失败

**错误信息**:
```
❌ PostgreSQL 连接失败（已重试 3 次）: password authentication failed
```

**排查步骤**:
1. 检查 `.env` 中的 `DB_PASSWORD` 是否正确
2. 确认 PostgreSQL 允许密码认证: 检查 `pg_hba.conf`
3. 测试连接: `psql -h localhost -U postgres -d nl2sql`

---

### ❌ 错误 5: Embedding API 调用失败

**错误信息**:
```
❌ 向量化失败（已重试 2 次）: Invalid API key
```

**排查步骤**:
1. 检查 `.env` 中的 `DASHSCOPE_API_KEY` 是否正确
2. 确认 API Key 是否有效: 访问 https://dashscope.console.aliyun.com/apiKey
3. 检查网络是否能访问通义千问服务

---

### ❌ 错误 6: 列不存在

**错误信息**:
```
❌ SQL 执行失败: column "company_name" does not exist
```

**解决方法**:
检查 `dim_tables.yaml` 中配置的 `embedding_col` 是否存在于该表中。

---

### ⚠️ 警告: 数据重复

**症状**:
每次运行不带 `--clean` 参数时，数据量翻倍。

**原因**:
当前版本的 DimValueLoader 不支持真正的增量更新，重复运行会追加数据。

**解决方法**:
使用 `--clean` 参数完全重建数据：
```bash
python -m src.metaweave.cli.main load \
  --type dim_value \
  --config configs/metaweave/loader_config.yaml \
  --clean
```

## 性能优化建议

### 1. 调整批量大小

编辑 `configs/metaweave/loader_config.yaml`:

```yaml
loaders:
  dim_value:
    batch_size: 200  # 默认 100，可根据内存调整到 200-500
```

**效果**:
- 批量越大，Embedding API 调用次数越少
- 但占用内存更多

### 2. 调整 Embedding 并发

编辑 `configs/metaweave/metadata_config.yaml`:

```yaml
llm:
  langchain_config:
    async_concurrency: 20  # 默认 10，可提升到 20-50
```

**效果**:
- 并发越高，向量化速度越快
- 但可能触发 API 限流

### 3. 过滤空值和重复值

DimValueLoader 已自动实现：
- `WHERE col IS NOT NULL AND LENGTH(BTRIM(CAST(col AS TEXT))) > 0`
- `SELECT DISTINCT` 去重

无需额外配置。
