# MetaWeave 日志模块设计文档

## 文档版本
- **版本**: v1.0
- **日期**: 2025-11-29
- **状态**: 设计阶段

## 一、设计目标

### 1.1 核心目标
1. **独立性**: 与主项目 nl2sql 模块完全隔离，支持将来独立提取
2. **可追溯性**: 每个 Step 执行过程完整记录，便于问题定位
3. **调试友好**: DEBUG 模式下可详细追踪字段匹配过程
4. **一致性**: 与现有 nl2sql 模块日志风格保持一致

### 1.2 具体需求
- 支持通过 YAML 配置日志输出目录
- 支持不同日志级别（INFO/DEBUG/ERROR）
- 支持按模块分离日志文件
- 支持日志文件滚动（避免单文件过大）
- **关键**：DEBUG 模式下记录字段匹配详情，快速定位关联失败原因

### 2.2 Logger 命名规范
- **命名空间前缀**: `metaweave`（独立于 `nl2sql`）
- **模块 Logger 命名**: `metaweave.<module_name>`
- **子模块 Logger 命名**: `metaweave.<module>.<submodule>`

**示例**:
```python
metaweave.metadata              # Step 2 元数据生成
metaweave.metadata.connector    # 数据库连接器
metaweave.metadata.generator    # 元数据生成器
metaweave.relationships         # Step 3 关系发现
metaweave.relationships.candidate_generator  # 候选生成器
metaweave.relationships.scorer  # 评分器
metaweave.cql_generator         # Step 4 CQL 生成
metaweave.loader                # Step 7.1 数据加载
```

### 2.3 日志级别策略
| 输出渠道 | 日志级别 | 说明 |
|---------|---------|------|
| **控制台** | INFO | 显示关键进度信息，用户友好 |
| **文件** | DEBUG | 记录详细执行过程，用于调试 |
| **错误** | ERROR + 异常栈 | 记录完整错误信息，便于定位问题 |

## 三、工具函数设计

### 3.1 核心函数
文件位置: `src/metaweave/utils/logger.py`

```python
"""MetaWeave 日志工具模块

提供独立的日志配置和获取函数，支持将来模块独立提取。
"""

import logging
import logging.config
from pathlib import Path
from typing import Optional
import yaml


def setup_metaweave_logging(config_path: Optional[str] = None) -> None:
    """初始化 MetaWeave 日志系统

    从 YAML 配置文件加载日志配置。如果未提供配置文件路径，
    使用默认配置 configs/metaweave/logging.yaml。

    Args:
        config_path: 日志配置文件路径（可选）

    Raises:
        FileNotFoundError: 配置文件不存在
    """
    if config_path is None:
        config_path = "configs/metaweave/logging.yaml"

    config_file = Path(config_path)
    if not config_file.exists():
        raise FileNotFoundError(f"日志配置文件不存在: {config_path}")

    with open(config_file, "r", encoding="utf-8") as f:
        cfg = yaml.safe_load(f)

    # 确保所有日志文件的目录存在
    for handler_name, handler_cfg in (cfg.get("handlers", {}) or {}).items():
        filename = handler_cfg.get("filename")
        if filename:
            log_path = Path(filename)
            log_path.parent.mkdir(parents=True, exist_ok=True)

    logging.config.dictConfig(cfg)


def get_metaweave_logger(module_name: str) -> logging.Logger:
    """获取 MetaWeave 模块日志器

    Args:
        module_name: 模块名（如 "metadata", "relationships"）

    Returns:
        logging.Logger: 日志器实例（命名: metaweave.<module_name>）

    Examples:
        >>> logger = get_metaweave_logger("metadata")
        >>> logger.info("开始生成元数据")

        >>> logger = get_metaweave_logger("relationships.scorer")
        >>> logger.debug("计算候选评分")
    """
    logger_name = f"metaweave.{module_name}"
    return logging.getLogger(logger_name)
```

### 3.2 使用示例
```python
# 在各个模块文件顶部导入
from src.metaweave.utils.logger import get_metaweave_logger

# 创建模块级别的 logger
logger = get_metaweave_logger("metadata")  # → metaweave.metadata

# 使用日志
logger.info("开始生成元数据")
logger.debug(f"正在处理表: {table_name}")
logger.error(f"处理失败: {error}", exc_info=True)
```

### 4.1 YAML 配置文件

```yaml

  standard:
    format: "%(asctime)s - %(levelname)s - %(name)s - %(message)s"
    datefmt: "%Y-%m-%d %H:%M:%S"

  detailed:
    format: "%(asctime)s - %(levelname)s - %(name)s - [%(filename)s:%(lineno)d] - %(message)s"
    datefmt: "%Y-%m-%d %H:%M:%S"

# 日志处理器定义
handlers:
  # 控制台输出（INFO 级别）

    class: logging.StreamHandler
    level: INFO
    formatter: standard
    stream: ext://sys.stdout

    class: logging.handlers.RotatingFileHandler
    level: DEBUG
    formatter: detailed

    backupCount: 5
    encoding: utf-8

  # Step 3: 关系发现日志
  relationships_file:
    class: logging.handlers.RotatingFileHandler
    level: DEBUG
    formatter: detailed
    filename: logs/metaweave/relationships.log
    maxBytes: 10485760  # 10MB
    backupCount: 5
    encoding: utf-8

  # Step 4: CQL 生成日志
  cql_generator_file:
    class: logging.handlers.RotatingFileHandler
    level: DEBUG
    formatter: detailed

    backupCount: 3
    encoding: utf-8

  # Step 7.1: 数据加载日志
  loader_file:
    class: logging.handlers.RotatingFileHandler
    level: DEBUG
    formatter: detailed
    filename: logs/metaweave/loader.log
    maxBytes: 5242880   # 5MB
    backupCount: 3
    encoding: utf-8

# Logger 配置
loggers:
  # Step 2: 元数据生成模块
  metaweave.metadata:
    level: DEBUG
    handlers: [console, metadata_file]
    propagate: false

  # Step 3: 关系发现模块
  metaweave.relationships:
    level: DEBUG
    handlers: [console, relationships_file]
    propagate: false

  # Step 4: CQL 生成模块
  metaweave.cql_generator:
    level: DEBUG
    handlers: [console, cql_generator_file]
    propagate: false

  # Step 7.1: 数据加载模块
  metaweave.loader:
    level: DEBUG
    handlers: [console, loader_file]
    propagate: false

# 根 logger（未匹配的 logger）
root:
  level: WARNING
  handlers: [console]
```

### 4.2 配置说明
1. **日志文件大小限制**:
   - relationships.log: 10MB（关系发现日志较多）
   - 其他: 5MB
2. **历史文件保留数量**:
   - relationships.log: 5 个历史文件
   - 其他: 3 个历史文件
3. **格式化器**:
   - `standard`: 简洁格式，用于控制台
   - `detailed`: 详细格式（含文件名和行号），用于文件

#### 5.2.2 候选生成器 (core/relationships/candidate_generator.py)
**关键**: 这是 DEBUG 模式下最重要的日志，用于排查字段匹配失败问题

```python
from src.metaweave.utils.logger import get_metaweave_logger

logger = get_metaweave_logger("relationships.candidate_generator")

class CandidateGenerator:
    def _generate_composite_candidates(self, tables):
        logger.debug("=" * 60)
        logger.debug("开始生成复合键候选")
        logger.debug("=" * 60)

        for source_name, source_table in tables.items():
            source_combinations = self._collect_source_combinations(source_table)

            logger.debug(f"\n源表: {source_name}")
            logger.debug(f"  → 找到 {len(source_combinations)} 个复合键组合")

            for combo in source_combinations:
                logger.debug(f"  → 组合: {combo['columns']} (类型: {combo['type']})")

                for target_name, target_table in tables.items():
                    if target_name == source_name:
                        continue

                    # 查找目标列
                    target_columns = self._find_target_columns(
                        combo['columns'], source_table, target_table, combo['type']
                    )

                    if target_columns:
                        logger.debug(f"    ✓ 找到匹配: {target_name}")
                        logger.debug(f"      源列: {combo['columns']}")
                        logger.debug(f"      目标列: {target_columns}")

                        # 记录详细的相似度信息
                        self._log_similarity_details(
                            combo['columns'], target_columns,
                            source_table, target_table
                        )
                    else:
                        logger.debug(f"    ✗ 未找到匹配: {target_name}")
                        logger.debug(f"      源列: {combo['columns']}")
                        logger.debug(f"      原因: 名称相似度或类型兼容性不满足阈值")

    def _log_similarity_details(self, source_cols, target_cols,
                                source_table, target_table):
        """记录详细的相似度计算结果（仅 DEBUG 模式）"""
        if not logger.isEnabledFor(logging.DEBUG):
            return

        logger.debug("      详细匹配信息:")

        total_name_sim = 0
        total_type_compat = 0

        for src_col, tgt_col in zip(source_cols, target_cols):
            # 计算名称相似度
            name_sim = self._calculate_name_similarity(src_col, tgt_col)

            # 计算类型兼容性
            src_type = source_table['column_profiles'][src_col]['data_type']
            tgt_type = target_table['column_profiles'][tgt_col]['data_type']
            type_compat = self._get_type_compatibility_score(src_type, tgt_type)

            total_name_sim += name_sim
            total_type_compat += type_compat

            logger.debug(f"        {src_col} ({src_type}) → {tgt_col} ({tgt_type})")
            logger.debug(f"          名称相似度: {name_sim:.4f}")
            logger.debug(f"          类型兼容性: {type_compat:.4f}")

        avg_name_sim = total_name_sim / len(source_cols)
        avg_type_compat = total_type_compat / len(source_cols)

        logger.debug(f"      平均名称相似度: {avg_name_sim:.4f} (阈值: {self.min_name_similarity})")
        logger.debug(f"      平均类型兼容性: {avg_type_compat:.4f} (阈值: {self.min_type_compatibility})")

        if avg_name_sim >= self.min_name_similarity and avg_type_compat >= self.min_type_compatibility:
            logger.debug(f"      ✓ 候选生成成功")
        else:
            logger.debug(f"      ✗ 候选生成失败（阈值不满足）")
```

#### 5.2.4 决策引擎 (core/relationships/decision_engine.py)
```python
from src.metaweave.utils.logger import get_metaweave_logger

logger = get_metaweave_logger("relationships.decision_engine")

class DecisionEngine:
    def decide(self, scored_candidates):
        logger.debug(f"开始决策: {len(scored_candidates)} 个候选")

        accepted = []
        rejected = []

        for candidate in scored_candidates:
            score = candidate['composite_score']
            src = candidate['source']['table_info']['table_name']
            tgt = candidate['target']['table_info']['table_name']

            if score >= self.accept_threshold:
                accepted.append(candidate)
                logger.debug(f"  ✓ 接受: {src} → {tgt} (评分: {score:.4f})")
            else:
                rejected.append(candidate)
                logger.debug(f"  ✗ 拒绝: {src} → {tgt} (评分: {score:.4f}, 阈值: {self.accept_threshold})")

        logger.info(f"决策完成: 接受 {len(accepted)}/{len(scored_candidates)} 个关系")
        return accepted
```

```python
            logger.info("清空 Neo4j 数据库")
            self.neo4j.execute_query("MATCH (n) DETACH DELETE n")
            logger.info("✓ 数据库已清空")

        # 读取 CQL 文件
        logger.info(f"读取 CQL 文件: {self.cql_file}")
        statements = self._parse_cql_file()
        logger.info(f"  → 解析语句: {len(statements)} 条")

        # 批量执行
        logger.info("批量加载数据")
        batch_size = 100
        for i in range(0, len(statements), batch_size):
            batch = statements[i:i+batch_size]
            batch_num = i // batch_size + 1
            total_batches = (len(statements) + batch_size - 1) // batch_size

            logger.info(f"  → 批次 [{batch_num}/{total_batches}]: {len(batch)} 条语句")

            try:
                for stmt in batch:
                    logger.debug(f"    执行: {stmt[:100]}...")
                    self.neo4j.execute_query(stmt)
            except Exception as e:
                logger.error(f"  ✗ 批次加载失败: batch={batch_num}", exc_info=True)
                raise

        # 验证
        logger.info("验证加载结果")
        stats = self._get_graph_stats()
        logger.info(f"  → 表节点: {stats['table_count']} 个")
        logger.info(f"  → 列节点: {stats['column_count']} 个")
        logger.info(f"  → JOIN_ON 关系: {stats['join_count']} 个")

        logger.info("=" * 80)
        logger.info(f"CQL 加载完成: 耗时 {duration:.2f}s")
        logger.info("=" * 80)
```

|------|------|------|
