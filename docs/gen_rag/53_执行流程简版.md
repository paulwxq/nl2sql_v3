# 53 简易执行指南（从生成到写入 Neo4j）

适用于改造后的 `metadata` CLI。使用项目内虚拟环境示例命令：

```bash
.venv-win\Scripts\python -m src.metaweave.cli.main metadata --config configs/metaweave/metadata_config.yaml --step <step> [options]
```

## 1. 准备 db_domains.yaml
- 位置：`configs/metaweave/db_domains.yaml`
- 必填：`database.description`
- 可选：`domains`（若为空，可用 `--generate-domains` 自动生成）

## 2. 生成 domains（可选，可单独运行）
- 仅需生成 domain 列表时执行，执行后退出主流程（无需指定 --step）。
```bash
--generate-domains
# 带 md 摘要（默认最多 100 个文件，模式默认 name_comment）
--generate-domains --md-context \
  [--md-context-dir output/metaweave/metadata/md] \
  [--md-context-mode name|name_comment|full] \
  [--md-context-limit 100]
```
行为：读取 `database.description`（必填，可附加 md 摘要）→ 调用 LLM 生成 domains → 写回 yaml（自动补 `_未分类_`，保留其他字段）。

## 3. 生成表画像与业务属性（json_llm）
- 只生成业务类型（table_category）：
```bash
--step json_llm
```
- 同时生成业务主题（table_domains），使用全部/指定 domain：
```bash
--step json_llm --domain         # 等价于 --domain all
--step json_llm --domain "A,B"   # 仅使用指定 domain
```
- 相关可选项：
  - `--domains-config <path>` 自定义 db_domains.yaml 路径
  - 若 `domains` 为空且使用 `--domain`，会报错提示先生成

## 4. 按 domain 发现关系（rel_llm）
- 全量（不分域）：
```bash
--step rel_llm
```
- 按域内关系：
```bash
--step rel_llm --domain          # 或 --domain all
--step rel_llm --domain "A,B"
```
- 域内 + 跨域：
```bash
--step rel_llm --domain "A,B" --cross-domain
--step rel_llm --domain all --cross-domain
```
- 仅跨域补充（基于已有域内结果）：
```bash
--step rel_llm --cross-domain
```
- 前置要求：使用 `--domain` / `--cross-domain` 时，json_llm 输出必须含 `table_domains`，且 db_domains.yaml 存在且 domains 非空。

## 5. 生成并写入 CQL（cql_llm / cql）
- LLM 流程（读取 json_llm 输出，写入 table_category/table_domains 到 Neo4j）：
```bash
--step cql_llm
```
- 规则流程（读取 json 输出）：
```bash
--step cql
```
- 输出目录默认：`output/metaweave/metadata/cql/`

## 6. 典型全流程示例
1) （可选）生成 domains 列表  
```bash
--generate-domains --md-context
```
2) 生成表画像 + 业务属性  
```bash
--step json_llm --domain
```
3) 按域发现关系（含跨域）  
```bash
--step rel_llm --domain all --cross-domain
```
4) 写入 Neo4j  
```bash
--step cql_llm
```

## 7. 常见校验与错误提示
- `--generate-domains` 与 `--domain` 互斥，不能同用。
- 使用 `--domain` / `--cross-domain` 时：
  - `db_domains.yaml` 必须存在且 `domains` 非空。
  - 表 JSON 必须包含 `table_domains`，否则报错退出。
- `--domain` 无值时等价 `--domain all`（Click `flag_value="all"`）。
