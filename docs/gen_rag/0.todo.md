1.若你要进入 Phase 2，建议优先考虑 schema 粒度输出与评分阶段 DB 采样的进一步优化（例如列/元组采样接口 + TABLESAMPLE），但这不影响当前模块的正确性与规范一致性。
2.多字段关联的发现，主要是源表中如果有逻辑复合主键，就要考虑启用这个。
  - 是否要考虑当源表没有主键时，逻辑主键存在并且是复合主键，此时要考虑启用复合主键的发现。
    - 也要强制启用主动名称发现，在目标表中寻找同名字段，类型兼容性也要考虑。
    - 在此基础上再启用非同名复合键的发现。
3.要排除一些特殊情况，比如自增长的id整形字段。这些字段每个表中都会存在，并且是逻辑主键。
5.要考虑使用LLM来判断的可行性，或者来校验的可信性。
