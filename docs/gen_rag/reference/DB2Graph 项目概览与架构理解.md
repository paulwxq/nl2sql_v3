# DB2Graph 项目概览与架构理解

## 项目定位与目标
- **DB2Graph** 旨在自动化从 PostgreSQL 中抽取元数据与数据画像，发现潜在的表间关系，并以图数据库/文件形式输出。
- MVP 阶段采用"确定性流水线 + 可选 LLM 增强"的策略，确保在无外部 API 时也能稳定运行。
- 目标输出既包括写入 Neo4j 的节点/关系，也支持生成 Cypher 脚本和分析报告，便于审阅与复用。

## 流程编排概览
整体流程由 `src/agent/orchestrator.py` 中的 Orchestrator 协调，关键阶段如下：
1. **读取元数据**：通过 MetadataReader 使用异步连接器采集表结构、主外键、索引等信息，并提取已存在的外键约束。
2. **数据采样与画像**：DataSampler 对候选列执行统计分析（唯一度、样本值、空值率等），支持字段标准化配置。
3. **候选生成**：CandidateGenerator 依据列名相似度、类型兼容性、模式匹配等规则生成单列和复合键关系候选，排除已存在的外键。
4. **度量评估**：MetricsEvaluator 计算包含率、Jaccard系数、唯一度并合成综合评分，同时标注置信度等级。
5. **关系决策**：根据配置的阈值和决策规则挑选最终关系，支持复合关系优先、子集抑制等策略。
6. **结果输出**：
   - CypherGenerator 生成 Cypher 语句，可落盘至 output/cypher。
   - GraphWriter 可直连 Neo4j 写入节点与关系，并维护约束。
   - 报告模块生成 JSON 与 Markdown 摘要，写入 output/reports。

## 核心模块拆解
| 模块 | 位置 | 职责 |
| ---- | ---- | ---- |
| 配置中心 | `src/config/settings.py` | 基于 Pydantic 定义数据库、采样、阈值、Embedding、日志等配置，支持 `${ENV}` 环境变量替换；支持 tables.include/exclude 通配过滤与 discovery.type_compatibility.groups 配置 |
| 数据模型 | `src/core/models.py` | 定义表/列元数据、关系候选、评分度量、最终关系、分析报告等数据结构，支持数值精度控制 |
| PostgreSQL 连接 | `src/connectors/` | 异步连接池（connection_manager）+ 异步适配器（postgresql），支持批量并发查询 |
| 元数据读取 | `src/tools/metadata_reader.py` | 并发读取表信息、提取已有外键、统计数据类型分布；支持 include/exclude 过滤并在统计缺失时回退精确行数 |
| 数据采样 | `src/tools/data_sampler.py` | 批量列画像分析、采样值收集、包含率优化计算、复合键值估算，支持数据标准化规则 |
| 候选生成 | `src/tools/candidate_generator.py` | 生成单列候选、复合键候选、自引用候选；支持确定性名称匹配和可选的向量相似度计算；利用索引列与类型兼容分组提升筛选质量 |
| 度量评估 | `src/tools/metrics_evaluator.py` | 计算包含率、Jaccard系数、唯一度等指标，合成综合评分；支持单列和复合键关系的评估 |
| 关系决策 | `src/tools/metrics_evaluator.py` | 基于评分排序、去重、应用决策规则（如 PK↔PK 封顶、子集抑制）组装最终 Relationship |
| 图写入 | `src/tools/graph_writer.py` | 连接 Neo4j 并写入表节点与关系边，维护唯一约束，返回写入统计 |
| Cypher 导出 | `src/tools/cypher_generator.py` | 生成完整的 Cypher 脚本，包含表节点创建、关系创建、统计信息 |
| 向量相似度 | `src/tools/embedding_similarity_service.py` | 可选的向量模型服务，支持批量计算列名语义相似度，可配置 Qwen/OpenAI 等提供商 |
| CLI 入口 | `src/main.py` | Click CLI 命令，支持运行分析、测试连接、查看 LLM 统计、试运行模式 |

## 配置体系亮点
- **环境变量替换**：load_config 递归解析 `${VAR:default}`，支持数据库连接信息的环境变量配置。
- **表过滤规则**：tables.include 和 tables.exclude 二选一，支持通配符模式（如 `sales_*`），在启动时校验互斥配置。
- **权重与阈值**：thresholds.weights 控制综合评分加权（包含率、名称相似度、类型兼容性、Jaccard、唯一度）；decision 设置置信度分级与决策规则；output.neo4j.min_score 控制写出门槛。
- **类型兼容性**：discovery.type_compatibility.groups 完全可配置，支持自定义数据类型分组策略。
- **数据标准化**：normalization 可按规则链式处理列值（trim/lower/去前导零等），提升匹配准确度。
- **复合键配置**：discovery.composite 控制复合键候选生成，支持配置候选来源（主键/唯一键/外键/索引）、最小相似度阈值、是否允许混合来源等。
- **决策规则**：thresholds.decision 支持复合关系优先、子集抑制、PK↔PK封顶、同表对关系数量限制等多种策略。
- **Embedding 配置**：可选择启用向量模型增强名称相似度计算，支持 Qwen/OpenAI 等提供商，配置批次大小、超时、重试策略。

## 关系发现算法要点
- **名称相似度**：采用三层策略（确定性规则 → 向量模型 → 编辑距离），支持同义词配置和通用标识符过滤（如 id, code）。
- **类型兼容性**：基于可配置的类型分组，完全相同返回 1.0，同组返回 0.8，不同组返回 0.0。
- **包含率优化**：采用"采样源表 + 批量 IN 查询目标表"的方式，避免全表 JOIN，显著提升大表性能。
- **复合键匹配**：支持列顺序的排列组合匹配，选择名称相似度和类型兼容性最优的映射方案。
- **综合评分**：默认权重为 包含率 45%、名称相似度 20%、类型兼容性 20%、Jaccard 15%、唯一度 10%，完全可配置。
- **决策规则**：支持按评分排序、去重、复合优先、子集抑制、PK↔PK 封顶等多种策略，确保关系质量。

## 输出与监控
- **分析报告**：JSON 格式记录完整运行信息（配置、统计、所有关系及度量）；Markdown 摘要按置信度分组展示 Top 关系，配置可调最小展示分与 Top-K。
- **Cypher 脚本**：生成完整的 Neo4j 导入语句，包含表节点、关系边、所有度量指标、统计信息。
- **日志系统**：基于 Loguru，支持控制台、文件、JSON 三种输出方式，可配置日志级别、轮转策略、保留时间。
- **Neo4j 写入统计**：返回创建/更新的节点与关系数量，支持清理现有数据选项。
- **性能监控**：记录各步骤耗时、并发任务数、数据量等关键指标。

## 近期更新（2025-09）
- **表过滤增强**：tables.include / tables.exclude 支持通配符列表（如 `public.sales_*`），并在加载时校验互斥配置，避免冲突。
- **元数据增强**：统计信息缺失或返回 -1 时自动回退执行精确 COUNT(*)；新增 has_index 标记与完整的索引元数据。
- **索引列参与**：候选生成允许有索引的列直接参与筛选，复合索引按列拆分后逐个参与候选匹配。
- **类型兼容配置化**：discovery.type_compatibility.groups 完全迁移到配置文件，支持在 YAML 中自定义和扩展分组策略。
- **复合键完整支持**：实现了完整的复合键候选生成、评估、决策流程，支持 2-4 列的复合键关系发现。
- **决策规则丰富**：新增复合关系优先、子集抑制、PK↔PK 封顶、同表对数量限制等决策策略配置。
- **数值精度统一**：所有评分指标统一保留两位小数，提升报告可读性和一致性。

## 当前限制与扩展空间
- **数据库支持**：PostgreSQL 是唯一支持的源数据库；Neo4j 为主要图数据库输出。未来可扩展支持 MySQL、Oracle 等。
- **图模型**：语义图模型（semantic）仍为预留入口，当前实现集中在轻量级模型（lightweight）。
- **复合键限制**：当前支持 2-4 列的复合键，可通过配置 discovery.composite.max_columns 调整。
- **向量模型**：Embedding 功能为可选增强，核心算法不依赖外部 API，确保离线可用性。
- **LLM 增强**：预留了 LLM 接口用于业务语义理解，但核心流程不依赖 LLM，遵循"增强不降权"原则。
- **自引用关系**：仅在启用 pattern_matching 策略时生成，基于列名模式识别（parent_id、manager_id 等）。

## 性能特性
- **异步并发**：元数据读取、列画像分析、关系评估均采用 asyncio 批量并发，可配置并发批次大小。
- **连接池管理**：使用 asyncpg 连接池，避免频繁建立连接，支持配置池大小（默认 2-10）。
- **采样策略**：通过配置 sampling.ratio 和 sampling.max_rows 控制采样规模，平衡准确性与性能。
- **查询优化**：包含率计算使用优化算法（采样 + 批量 IN 查询），避免全表 JOIN，支持大规模数据库。
- **批量处理**：关系评估采用批量异步处理，默认批次大小为 3，可通过配置调整。

## 典型使用场景
- **数据库逆向工程**：自动发现未定义的外键关系，补全数据库设计文档。
- **数据血缘分析**：构建表间关系图谱，分析数据流向和依赖关系。
- **知识图谱构建**：将关系型数据库的表结构转换为图数据库，支持图查询和分析。
- **数据治理**：识别数据关联关系，辅助数据质量评估和数据标准制定。
- **迁移辅助**：在数据库迁移或重构时，自动识别表间依赖关系，降低风险。
