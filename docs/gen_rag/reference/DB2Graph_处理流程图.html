📊 核心数据流

  • 输入：PostgreSQL 数据库（元数据 + 数据样本）
  • 中间产物：
    • TableMetadata：表元数据字典（步骤1）
    • ColumnProfile：列数据画像字典（步骤2）
    • RelationshipCandidate：候选关系列表（步骤3）
    • RelationshipMetrics：关系度量列表（步骤4）
    • Relationship：最终关系列表（步骤5）
  • 输出：
    • Neo4j 图数据库（表节点 + 关系边）
    • Cypher 脚本文件（output/cypher/*.cypher）
    • JSON 分析报告（output/reports/*.json）
    • Markdown 摘要（output/reports/*.md）

⚡ 性能优化点

  • 异步并发：元数据读取、列画像分析、关系评估均采用 asyncio.gather 批量并发
  • 连接池：使用 asyncpg 连接池，避免频繁建立连接
  • 包含率优化：采用"采样 + 批量 IN 查询"替代全表 JOIN
  • 批量处理：关系评估每批处理 3 个（可配置），避免连接池耗尽
  • 采样策略：可配置采样比例和最大行数，平衡准确性与性能

🎯 核心算法

  • 类型兼容性：基于配置的类型分组，完全相同=1.0，同组=0.8，不同组=0.0
  • 名称相似度：确定性规则 → 向量模型（可选）→ 编辑距离（回退）
  • 包含率：从源表采样值，批量查询目标表存在性，计算比例
  • 综合评分：0.45×包含率 + 0.20×名称 + 0.20×类型 + 0.15×Jaccard + 0.10×唯一度
  • 复合键匹配：排列组合所有可能的列映射，选择相似度最高的方案
