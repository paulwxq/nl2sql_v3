### 核心概念：三种集成方式

LangGraph 1.0.x提供了**三种方式**让子图与父图交互：

### 方式1: **State继承（隐式共享）**

```python
class ParentState(TypedDict):
    question: str
    sql: str
    result: Dict

# 子图直接使用父图的State类型
sub_graph = StateGraph(ParentState)
# ✅ 子图可以读写父图的所有字段
# ❌ 没有封装，状态混在一起
```

**特点：**

- 子图**直接修改**父图的State
- 所有字段共享
- 简单但不推荐（状态污染）

### 方式2: **独立State + Input/Output Schema（推荐）**

LangGraph 1.0引入的新特性：

```python
from langgraph.types import Command

# 子图有自己的State
class SubState(TypedDict):
    question: str  # input
    vector_results: List  # 内部字段
    sql: str  # output

# 父图的State
class ParentState(TypedDict):
    question: str
    generated_sql: str

# 关键：定义input/output映射
parent_graph.add_node(
    "sql_gen",
    sub_graph,
    input=["question"],  # 从父图传这些字段给子图
    output=["sql"]  # 子图的sql字段映射到父图的哪里？
)
```

在1.0.x中，有两种方式：

### A. 字段名匹配（同名自动映射）

```python
class SubState(TypedDict):
    question: str
    generated_sql: str  # 与父图同名

class ParentState(TypedDict):
    question: str
    generated_sql: str

parent_graph.add_node(
    "sql_gen",
    sub_graph,
    input=["question"],
    output=["generated_sql"]  # 自动映射
)
```

### B. 使用Command显式映射（1.0新特性）

```python
# 子图返回Command
def final_node(state: SubState):
    return Command(
        update={
            "generated_sql": state["sql"],  # 显式指定映射
            "success": state["is_valid"]
        }
    )

# 父图配置
parent_graph.add_node("sql_gen", sub_graph, input=["question"])
# Command会自动处理output映射
```

方式3: **Wrapper节点（最灵活）**

```python
def sql_wrapper(state: ParentState):
    # 手动调用子图
    result = sub_graph.invoke({"question": state["question"]})
    
    # 手动提取需要的字段
    return {
        "generated_sql": result["sql"],
        "metadata": {"tokens": result["token_count"]}
    }

parent_graph.add_node("sql_gen", sql_wrapper)
```

# 父图
class ParentState(TypedDict):

class SubState(TypedDict):

两种方式的对比

```python
# 方式1: 用input/output（简单）
parent_graph.add_node(
    "sql_gen", 
    sub_graph,
    output=["sql"]
)

# 方式2: 用wrapper（更灵活，可以改字段名）
def wrapper(state):
    result = sub_graph.invoke({"question": state["question"]})
    return {"sql": result["sql"]}

parent_graph.add_node("sql_gen", wrapper)
```

**结论：能用input/output就用，不需要wrapper。**

wrapper的唯一好处是可以改字段名：

```python
# 子图返回"generated_sql"，父图字段是"sql"
def wrapper(state):
    result = sub_graph.invoke(...)
    return {"sql": result["generated_sql"]}  # 改名
```

另外一个Wrapper 的例子：

```python
# ===== 子图 =====
class SQLGenState(TypedDict):
    question: str  # input
    # 内部字段（父图不需要知道）
    vector_results: List
    neo4j_results: List
    retry_count: int
    # output
    sql: str
    success: bool

sql_graph = StateGraph(SQLGenState)
# ... 构建子图

# ===== 父图 =====
class ParentState(TypedDict):
    user_question: str
    current_sql: str
    sql_success: bool

parent_graph = StateGraph(ParentState)

# 方式A: 如果字段名不匹配，用wrapper
def sql_node(state: ParentState):
    result = sql_graph.invoke({"question": state["user_question"]})
    return {
        "current_sql": result["sql"],
        "sql_success": result["success"]
    }

parent_graph.add_node("sql_gen", sql_node)

# 方式B: 如果字段名匹配，直接用
# （需要改ParentState的字段名为question, sql, success）
parent_graph.add_node(
    "sql_gen",
     sql_graph,
     input=["question"],
     output=["sql", "success"]
 )
```
