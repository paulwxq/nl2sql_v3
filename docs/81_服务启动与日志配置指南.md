# 81 服务启动与日志配置指南

## 1. 服务启动

### 1.1 启动顺序

**先启动 FastAPI 后端，再启动 Streamlit 前端。**

Streamlit 启动后会立即调用后端 API 加载历史会话列表，如果后端未就绪，侧边栏会显示"无法连接后端服务"。

### 1.2 启动 FastAPI 后端

在项目根目录下执行：

```bash
# 前台启动（开发调试用，Ctrl+C 停止）
uvicorn src.api.main:app --host 0.0.0.0 --port 8008 --reload

# 后台启动（终端关闭后仍运行）
nohup uvicorn src.api.main:app --host 0.0.0.0 --port 8008 --reload > logs/uvicorn.log 2>&1 &
```

- `--port 8008`：默认端口，可按需修改
- `--reload`：代码变更后自动重启，生产环境建议去掉

### 1.3 启动 Streamlit 前端

在项目根目录下执行：

```bash
# 前台启动
streamlit run frontend/app.py --server.port 8501

# 后台启动
nohup streamlit run frontend/app.py --server.port 8501 > logs/streamlit_server.log 2>&1 &
```

启动后浏览器访问 `http://localhost:8501`。

### 1.4 停止服务

```bash
# 方式一：通过进程名查找并停止
kill $(pgrep -f "uvicorn src.api.main")
kill $(pgrep -f "streamlit run frontend/app.py")

# 方式二：如果是当前终端的后台任务
jobs        # 查看后台任务编号
kill %1     # 停止对应编号的任务
```

### 1.5 端口配置

| 服务 | 默认端口 | 修改方式 |
|------|---------|---------|
| FastAPI | 8008 | 启动命令 `--port` 参数 |
| Streamlit | 8501 | 启动命令 `--server.port` 参数 |

如果修改了 FastAPI 端口，需要同步修改前端的后端地址。两种方式任选其一：

- 修改 `frontend/api_client.py` 中的 `API_BASE_URL` 默认值
- 设置环境变量：`export API_BASE_URL=http://localhost:<新端口>/api/v1`

## 2. 日志配置

项目有三套日志，分属不同进程，配置位置各不相同。

### 2.1 总览

| 日志 | 所属进程 | 配置位置 | 输出文件 |
|------|---------|---------|---------|
| 后端核心（NL2SQL 管线） | FastAPI | `src/configs/logging.yaml` | `logs/sql_subgraph.log` |
| FastAPI API 层 | FastAPI | `src/configs/logging.yaml` | `logs/fastapi.log` |
| Streamlit 前端 | Streamlit | `frontend/logger.py` | `logs/streamlit.log` |

### 2.2 后端核心 + FastAPI 日志

配置文件：`src/configs/logging.yaml`

```yaml
handlers:
  sql_subgraph_file:       # 后端核心管线
    filename: logs/sql_subgraph.log
    level: DEBUG            # ← 修改日志级别
    maxBytes: 5242880       # ← 修改单文件大小（字节）
    backupCount: 5          # ← 修改保留历史文件数

  fastapi_file:            # FastAPI API 层
    filename: logs/fastapi.log
    level: INFO             # ← 修改日志级别
    maxBytes: 5242880
    backupCount: 5

loggers:
  nl2sql:                  # 后端核心
    level: DEBUG            # ← 修改日志级别

  nl2sql.fastapi:          # FastAPI API 层
    level: INFO             # ← 修改日志级别
```

修改后重启 FastAPI 服务即可生效（如使用 `--reload` 则自动生效）。

### 2.3 Streamlit 前端日志

配置文件：`frontend/logger.py`

```python
streamlit_logger.setLevel(logging.INFO)  # ← 修改日志级别

fh = logging.FileHandler("logs/streamlit.log", encoding="utf-8")  # ← 修改输出路径

fh.setFormatter(
    logging.Formatter("%(asctime)s [%(levelname)s] %(message)s")  # ← 修改日志格式
)
```

Streamlit 前端是独立进程，无法导入 `src` 包，因此不使用 `logging.yaml`，直接在 Python 代码中配置。修改后重启 Streamlit 服务生效。
