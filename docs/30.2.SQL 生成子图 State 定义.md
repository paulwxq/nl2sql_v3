## SQL 生成子图 State 定义（最新）

- 源码位置：`src/modules/sql_generation/subgraph/state.py`
- 适用范围：v3 最新实现（以源码为准）

### 基类与继承
- 基类：`MessagesState`
- 继承字段：`messages: List[BaseMessage]`（LLM 交互消息，随流程累积）

### 字段分组与说明

- 输入字段
  - `query: str`：子查询文本
  - `query_id: str`：查询 ID
  - `dependencies_results: Dict[str, Any]`：依赖子查询的结果
  - `user_query: str`：用户原始自然语言问题
  - `parse_hints: Optional[Dict[str, Any]]`：解析提示（可由上游传入）

- 解析阶段
  - `parse_result: Optional[Dict[str, Any]]`：问题解析的结构化结果（内部 LLM 或外部注入）
  - `parsing_source: Optional[str]`：解析来源，取值 `"external" | "llm" | "disabled" | "fallback"`

- Schema 检索阶段
  - `schema_context: Optional[Dict[str, Any]]`：检索到的上下文聚合
    - 结构参考：
      - `tables: List[...]`：候选相关表（向量检索）
      - `columns: List[...]`：候选相关列（向量检索）
      - `join_plans: List[...]`：推荐 JOIN 路径（图检索/多 Base）
      - `table_cards: Dict[...]`：表卡片字典
      - `similar_sqls: List[...]`：历史成功 SQL 案例（兼容旧字段）
      - `dim_value_matches: List[...]`：维度值匹配结果（兼容旧字段）
      - `candidate_fact_tables: List[...]`、`candidate_dim_tables: List[...]`
      - `table_similarities: Dict[...]`、`dim_value_hits: List[...]`

- SQL 生成阶段
  - `generated_sql: Optional[str]`：当前迭代生成的 SQL
  - `iteration_count: int`：当前迭代次数（默认 0）

- 验证阶段
  - `validation_result: Optional[Dict[str, Any]]`：当次验证结果
    - 结构参考：
      - `valid: bool`
      - `errors: List[str]`
      - `warnings: List[str]`（性能警告）
      - `layer: str`（`"syntax" | "security" | "semantic" | "all_passed"`）
      - `explain_plan: Optional[str]`（EXPLAIN 计划）
  - `validation_history: List[Dict]`：所有历史验证记录（通过 reducer `add` 进行累加，避免共享可变默认值）

- 输出字段
  - `validated_sql: Optional[str]`：最终验证通过的 SQL（成功标志）
  - `error: Optional[str]`：错误信息（若成功会被对外屏蔽）
  - `error_type: Optional[str]`：错误类型（常见：`parsing_failed | schema_retrieval_failed | generation_failed | validation_failed`）
  - `execution_time: float`：本子图执行耗时（秒）

### 成功判定与输出提取（辅助）
- 成功判定：当 `validated_sql` 非空视为成功。
- 输出提取：`extract_output(state)` 返回的对外字段包括
  - `validated_sql`、`error`（成功时置空）、`error_type`（成功时置空）
  - `iteration_count`、`execution_time`、`schema_context`、`validation_history`

### 与流程拓扑的关联（简述）
- 子图主要节点：`question_parsing → schema_retrieval → sql_generation → validation`
- 失败与重试：`validation` 失败时依据配置与 `iteration_count` 决定是否回到 `sql_generation` 重试；
  - 若出现不可恢复错误（例如 `generation_failed` 等），直接结束为失败；
  - 解析失败（`parsing_failed`）在入口处终止流程。
