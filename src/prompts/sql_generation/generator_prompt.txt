# SQL 编写规范

## 表名规范
- 必须使用完整的 schema.table 格式：`public.fact_sales`
- 建议使用简短别名：`FROM public.fact_sales fs`

## 时间过滤规范
- 使用指定的时间列（见下方"时间列"部分）
- 时间范围使用半开区间：
  ```sql
  WHERE date_col >= '2023-01-01' AND date_col < '2023-04-01'
  ```
- 不要使用 BETWEEN，因为它是闭区间

## JOIN 规范
- 严格按照"JOIN 计划"中提供的 ON 条件模板
- 用实际表别名替换模板中的 SRC 和 DST
- 示例：模板 `SRC.store_id = DST.store_id`
  → 实际：`fs.store_id = ds.store_id`
- 优先使用 INNER JOIN，除非明确需要外连接

## 聚合规范
- GROUP BY 必须包含 SELECT 中的所有非聚合列
- ORDER BY 通常按聚合结果排序（如 `ORDER BY SUM(amount) DESC`）

## 性能优化
- 尽量使用索引字段作为过滤条件
- 避免在 WHERE 中对字段进行函数运算（如 `DATE(datetime_col)`）
- 优先使用等值条件而非范围条件
- 如果提供了维度值的主键映射，使用 `key_col = 'value'` 而非 `LIKE '%value%'`
